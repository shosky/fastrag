# 知识库系统端到端测试流程文档

> 基于 mock 数据层的完整功能测试指南。
> 所有数据持久化在内存 mock store 中，刷新页面后重置。

---

## 前置条件

- 已启动开发服务器：`npm run dev`
- 默认登录用户：管理员（拥有全部权限）
- 浏览器打开：`http://localhost:3000`

---

## 一、知识库生命周期

### 1.1 查看知识库列表

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 访问 `/knowledge` | 显示 5 个知识库卡片（产品/技术/客户案例/培训/个人笔记） |
| 2 | 点击左侧"产品文档"分类 | 列表只显示分类为"产品文档"的知识库 |
| 3 | 点击"技术文档"分类 | 列表切换为技术文档分类 |
| 4 | 再次点击已选中的分类 | 取消筛选，显示全部 |
| 5 | 在搜索框输入"产品" | 列表实时过滤，只显示名称含"产品"的知识库 |
| 6 | 点击标签筛选按钮 | 展开标签面板，显示所有知识库的标签 |
| 7 | 点击"产品"标签 | 列表过滤为含"产品"标签的知识库 |
| 8 | 切换到"我的知识库" Tab | 显示当前用户可访问的知识库（基于 ACL） |

### 1.2 创建知识库

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 点击"创建知识库"按钮 | 跳转到 `/knowledge/create` |
| 2 | 不填任何字段，直接点击"创建知识库" | 表单校验失败，提示"请输入知识库名称" |
| 3 | 输入名称"测试知识库A"，选择分类"技术文档" | 校验通过 |
| 4 | 输入描述"这是一个测试知识库" | 描述字段填充 |
| 5 | 输入标签"测试"回车，再输入"demo"回车 | 标签区域显示 2 个标签 |
| 6 | 点击"测试"标签的关闭按钮 | 标签被移除，只剩"demo" |
| 7 | 切换到"共享设置" Tab | 显示全局/部门/指定人三个共享卡片 |
| 8 | 点击"部门共享"卡片 | 弹窗显示部门列表，默认选中第一个部门 |
| 9 | 点击弹窗外任意位置 | 弹窗关闭，卡片保持选中状态 |
| 10 | 点击"全局共享"卡片 | 切换到全局共享，部门选择清空 |
| 11 | 切换到"高级配置" Tab | 显示嵌入模型下拉、解析策略说明、文件类型配置 |
| 12 | 切换到"检索设置" Tab | 显示 6 个配置分组（模式/基础/预处理/重排序/多路召回/上下文） |
| 13 | 点击底部"创建知识库"按钮 | 提示"创建成功"，跳转回列表页 |
| 14 | 确认列表中出现"测试知识库A" | 新知识库卡片显示，分类/标签正确 |

### 1.3 编辑知识库

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 点击"产品知识库"卡片 | 跳转到详情页，显示真实的知识库名称 |
| 2 | 点击"编辑"按钮 | 跳转到 `/knowledge/1/edit` |
| 3 | 确认表单回填了知识库的真实数据 | 名称/描述/标签/嵌入模型正确 |
| 4 | 修改名称为"产品知识库V2" | 字段更新 |
| 5 | 切换到"检索设置" Tab | 显示该知识库的检索配置 |
| 6 | 关闭"自动纠错"开关 | 开关状态切换 |
| 7 | 开启"多路召回" | 展开通道数量配置 |
| 8 | 点击底部"保存修改" | 提示"保存成功"，跳转回列表 |
| 9 | 再次进入编辑页 | 确认名称和检索配置已持久化 |

### 1.4 删除知识库

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 进入某知识库编辑页 | 表单正常加载 |
| 2 | 点击右侧"删除知识库"按钮 | 弹出确认对话框"确定要删除该知识库吗？" |
| 3 | 点击"取消" | 对话框关闭，无操作 |
| 4 | 再次点击"删除知识库" → 点击"确定" | 提示"删除成功"，跳转回列表 |
| 5 | 确认列表中该知识库已消失 | 列表数量减 1 |
