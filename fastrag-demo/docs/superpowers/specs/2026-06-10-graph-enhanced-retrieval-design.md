# 图谱增强检索设计方案

## 概述

本设计方案描述了如何将知识图谱与现有检索功能结合，通过查询扩展增强文本检索效果。采用轻量级方案，保持现有UI不变，用户无感知扩展过程。

## 背景

### 现有系统
- **检索设置**：支持向量搜索、全文搜索、混合搜索
- **知识图谱**：展示实体节点和关系边，支持节点搜索
- **检索测试**：基于文本的检索测试

### 目标
- 在检索时利用图谱的实体关系增强文本检索结果
- 扩展过程完全透明，用户只看到最终结果
- 保持现有UI/UX不变

## 架构设计

### 核心流程

```
用户查询
    ↓
NER实体识别
    ↓
图谱实体查询（1跳）
    ↓
查询词扩展
    ↓
文本检索（向量/全文/混合）
    ↓
返回增强结果
```

### 核心组件

1. **实体识别器（NER）**
   - 从用户查询中提取实体
   - 支持产品名、服务名、组织名等类型
   - 可使用预训练模型或规则匹配

2. **图谱查询器**
   - 根据实体在图谱中查找相关实体和关系
   - 限制查询深度为1跳
   - 支持缓存机制

3. **查询扩展器**
   - 将扩展的实体/关系转换为检索关键词
   - 过滤低相关性扩展词
   - 保持查询语义一致性

4. **检索引擎**
   - 执行增强后的文本检索
   - 支持现有的向量/全文/混合模式

## 数据流设计

### 输入
- 查询字符串：用户输入的检索内容
- 启用标志：是否启用图谱扩展（默认true）

### 处理流程

1. **实体识别**
   - 输入："小微ICT服务有哪些"
   - 输出：["小微ICT"]

2. **图谱查询**
   - 查询实体："小微ICT"
   - 返回关联实体：["智能组网", "机房托管", "极速专线", "综合布线"]

3. **查询扩展**
   - 原始查询："小微ICT服务有哪些"
   - 扩展查询："小微ICT服务有哪些 智能组网 机房托管 极速专线 综合布线"

4. **文本检索**
   - 使用扩展查询执行检索
   - 返回增强后的结果

### 输出
- 检索结果：与现有格式相同
- 相似度分数：可能因扩展而变化
- 来源信息：保持不变

## API设计

### 检索接口增强

**请求**：
```json
POST /api/knowledge/{id}/search
{
  "query": "小微ICT服务",
  "enableGraphExpansion": true,
  "topK": 10,
  "similarityThreshold": 0.0,
  "mode": "hybrid"
}
```

**响应**（保持现有格式）：
```json
{
  "results": [
    {
      "index": 1,
      "similarity": 0.85,
      "content": "...",
      "source": "文档名.pdf",
      "fileId": "file_xxx",
      "chunkIndex": 0
    }
  ],
  "total": 1
}
```

### 图谱扩展接口（内部使用）

**请求**：
```json
POST /api/knowledge/{id}/graph/expand
{
  "entity": "小微ICT",
  "depth": 1,
  "maxEntities": 10
}
```

**响应**：
```json
{
  "entities": [
    {"id": "1", "name": "智能组网", "type": "服务"},
    {"id": "2", "name": "机房托管", "type": "服务"}
  ],
  "relations": [
    {"source": "小微ICT", "target": "智能组网", "label": "提供"}
  ]
}
```

## 性能优化

### 缓存策略
- **实体识别缓存**：相同查询的实体识别结果缓存10分钟
- **图谱查询缓存**：相同实体的查询结果缓存5分钟
- **扩展查询缓存**：相同扩展查询的检索结果缓存5分钟

### 异步处理
- 图谱查询与文本检索并行执行
- 使用Promise.all等待两者完成
- 超时控制：图谱查询超时3秒，超时后使用原始查询

### 资源限制
- 扩展深度：最多1跳
- 扩展实体数：最多10个
- 扩展词长度：最多50个字符

## UI/UX设计

## 风险与缓解

### 风险1：实体识别准确性
- **影响**：扩展词不相关，降低检索质量
- **缓解**：使用高质量NER模型，设置置信度阈值

### 风险2：图谱查询延迟
- **影响**：整体检索延迟增加
- **缓解**：缓存机制、异步处理、超时控制

### 风险3：扩展词过多
- **影响**：查询语义模糊
- **缓解**：限制扩展实体数量，过滤低相关性词
