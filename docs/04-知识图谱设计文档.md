# 知识图谱设计文档

## 1. 功能概述

知识图谱将文档中的实体和关系提取出来，构建结构化的知识网络，支持图谱可视化和图谱增强检索。

## 2. 数据模型

### 2.1 数据库表 `kb_graph_index`

| 字段 | 类型 | 说明 |
|------|------|------|
| kb_id | VARCHAR(32) PK | 知识库ID |
| status | VARCHAR(16) | 状态(idle/building/completed/failed) |
| build_progress | INT | 构建进度(0-100) |
| entity_count | INT | 实体数量 |
| relation_count | INT | 关系数量 |
| last_built_at | DATETIME | 最后构建时间 |

### 2.2 Neo4j 存储

- **Entity 节点**: name, type, kbId
- **关系**: 源实体 → 目标实体 (关系类型)

## 3. API 接口

### 3.1 获取图谱数据
```
GET /api/kb/{kbId}/graph
```

**查询参数：**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| maxNodes | int | 否 | 最大节点数(默认100) |
| excludeChunks | boolean | 否 | 是否排除切片节点 |

### 3.2 获取图谱统计
```
GET /api/kb/{kbId}/graph/stats
```

### 3.3 获取索引状态
```
GET /api/kb/{kbId}/graph/index
```

### 3.4 构建图谱索引
```
POST /api/kb/{kbId}/graph/index/build
```

**请求体：**
```json
{
  "mode": "full",
  "fileIds": ["file1", "file2"]
}
```

### 3.5 获取构建状态
```
GET /api/kb/{kbId}/graph/index/build-status
```

### 3.6 获取图谱设置
```
GET /api/kb/{kbId}/graph/settings
```

### 3.7 保存图谱设置
```
PUT /api/kb/{kbId}/graph/settings
```

## 4. 图谱构建流程

```
文档上传 → 切片完成 → GraphBuildConsumer
                          ↓
                   LLM 提取实体关系
                          ↓
                   Neo4j 存储图谱
```
