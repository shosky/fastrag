# 解析策略设计文档

## 1. 功能概述

解析策略定义了不同文件类型的解析方式、切片参数和使用的 AI 模型。

## 2. 数据模型

### 2.1 数据库表 `kb_parse_strategy`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(32) PK | 策略ID |
| kb_id | VARCHAR(32) FK | 知识库ID |
| name | VARCHAR(128) | 策略名称 |
| description | VARCHAR(256) | 描述 |
| extensions | JSON | 适用扩展名数组 |
| parse_method | VARCHAR(16) | 解析方法(default/pdf/pptx/video/audio) |
| is_default | TINYINT | 是否默认策略 |
| advanced | JSON | 高级参数(切片长度/分隔符等) |
| llm_model | VARCHAR(128) | LLM模型(智能分段) |
| vlm_model | VARCHAR(128) | VLM模型(图片理解) |

## 3. API 接口

### 3.1 策略列表
```
GET /api/kb/{kbId}/parse-strategies
```

### 3.2 创建策略
```
POST /api/kb/{kbId}/parse-strategies
```

### 3.3 更新策略
```
PUT /api/kb/{kbId}/parse-strategies/{id}
```

### 3.4 删除策略
```
DELETE /api/kb/{kbId}/parse-strategies/{id}
```

### 3.5 设为默认
```
POST /api/kb/{kbId}/parse-strategies/{id}/set-default
```

### 3.6 获取单个策略
```
GET /api/kb/{kbId}/parse-strategies/{id}
```

### 3.7 按扩展名解析
```
GET /api/kb/{kbId}/parse-strategies/resolve?extension=.pdf
```

### 3.8 检测扩展名冲突
```
POST /api/kb/{kbId}/parse-strategies/conflicts
```

**请求体：**
```json
{
  "extensions": [".pdf", ".docx"],
  "excludeId": "strategy_id_to_exclude"
}
```

### 3.9 获取策略模板
```
GET /api/parse-strategy-templates
```

**响应：**
```json
{
  "code": 200,
  "data": [
    {"key": "auto", "name": "通用文档策略", "extensions": [".pdf", ".docx", ...], "parseMethod": "default"},
    {"key": "pdf", "name": "PDF 专用策略", "extensions": [".pdf"], "parseMethod": "pdf"},
    {"key": "pptx", "name": "PPT 专用策略", "extensions": [".pptx", ".ppt"], "parseMethod": "pptx"},
    {"key": "media", "name": "音视频策略", "extensions": [".mp3", ".wav", ...], "parseMethod": "audio"}
  ]
}
```
