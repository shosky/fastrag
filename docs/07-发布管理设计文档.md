# 发布管理设计文档

## 1. 功能概述

发布管理支持知识库的版本控制和审核流程，确保知识库变更经过审核后才上线。

## 2. 数据模型

### 2.1 数据库表 `kb_version`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(32) PK | 版本ID |
| kb_id | VARCHAR(32) FK | 知识库ID |
| version | INT | 版本号 |
| name | VARCHAR(128) | 版本名称 |
| publish_status | VARCHAR(16) | 状态(draft/pending_review/approved/rejected/published) |
| file_count | INT | 文件数量 |
| chunk_count | INT | 切片数量 |
| created_by | VARCHAR(32) | 创建者 |

### 2.2 数据库表 `kb_review_task`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(32) PK | 审核ID |
| kb_id | VARCHAR(32) FK | 知识库ID |
| version_id | VARCHAR(32) FK | 版本ID |
| applicant | VARCHAR(64) | 申请人 |
| reviewer | VARCHAR(64) | 审核人 |
| status | VARCHAR(16) | 状态(pending/approved/rejected) |
| comment | TEXT | 审核意见 |

## 3. API 接口

### 3.1 版本管理
```
GET    /api/kb/{kbId}/versions           # 版本列表
GET    /api/kb/{kbId}/versions/latest    # 最新版本
GET    /api/kb/{kbId}/versions/published # 已发布版本
POST   /api/kb/{kbId}/versions           # 创建版本
POST   /api/kb/{kbId}/versions/{id}/transition # 状态流转
```

### 3.2 审核管理
```
GET    /api/reviews           # 审核列表(可选参数: kbId)
POST   /api/reviews           # 提交审核
POST   /api/reviews/{id}/approve # 审核通过
POST   /api/reviews/{id}/reject  # 审核驳回
```

## 4. 状态流转

```
draft → pending_review → approved → published
                     ↘ rejected → draft
```
